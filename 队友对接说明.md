## 一、课标分级标签库使用

**文件**：`src/data/textbookDatabase.ts`

### 核心数据结构

```typescript
// 学段分级标准
interface SchoolStageStandard {
  stage: 1 | 2 | 3;    // 1=1-2年级, 2=3-4年级, 3=5-6年级
  grades: string;       // 年级范围
  name: string;         // 学段名称
  goal: string;         // 学习目标
  requirement: string;  // 阅读要求
  systemLevel: string;  // 对应系统层级
  reciteCount: number;  // 累计背诵篇数
  keySkills: string[];  // 核心能力
}

// 古诗文条目
interface TextbookPoem {
  id: string;                // 唯一ID，如 'pep-5b-08'
  title: string;             // 诗题
  author: string;            // 作者
  dynasty: string;           // 朝代
  grade: number;             // 年级 (1-6)
  semester: '上' | '下';     // 学期
  type: '古诗' | '词' | '文言文' | '蒙学' | '寓言';
  stage: 1 | 2 | 3;          // 学段
  keywords: string[];        // 核心关键词
  cadalMatchHints: string[]; // CADAL 匹配方向
  isExamKey: boolean;        // 是否为考试重点篇目
  fullText?: string;         // 原文（关键篇目）
}
```

### 读取与筛选

```typescript
import {
  textbookPoems,           // 完整 126 篇数据库
  getPoemsByGrade,         // 按年级筛选
  getPoemsByStage,         // 按学段筛选
  getExamKeyPoems,         // 获取考试重点篇目
  getPoemsByKeyword,       // 按关键词搜索
  getCADALMatchHints,      // 获取某篇的 CADAL 匹配方向
  schoolStageStandards,    // 三学段分级标准
  stageStats,              // 学段统计数据
  topAuthors,              // 高频作者排行
  allKeywords,             // 全量关键词词云
} from './data/textbookDatabase';
```

### 常用查询示例

```typescript
// 获取第一学段所有篇目
const stage1Poems = getPoemsByStage(1);   // 1-2年级，共 20+ 篇

// 获取所有考试重点篇目
const examPoems = getExamKeyPoems();       // 约 40+ 篇

// 搜索所有和"月亮"相关的诗
const moonPoems = getPoemsByKeyword('月亮');

// 获取某篇的 CADAL 检索方向
const hints = getCADALMatchHints('pep-5b-08'); // ['世说新语', '魏晋轶事', '童趣文言']
```

## 二、篇目关键词匹配逻辑

每首诗有两个用于 CADAL 古籍检索的关键字段：

### `keywords` — 核心关键词

主题标签，用于快速筛选和分类。示例：

```typescript
// 静夜思
{ keywords: ['月亮', '思乡', '李白'] }

// 杨氏之子
{ keywords: ['世说新语', '儿童', '机智', '魏晋'] }

// 田忌赛马
{ keywords: ['策略', '史记', '智慧'] }
```

### `cadalMatchHints` — CADAL 匹配方向

告诉 CADAL 检索接口应该搜索哪些古籍资源。示例：

```typescript
// 静夜思 → 查李白诗集 + 唐代思乡诗
{ cadalMatchHints: ['李白诗集', '唐代思乡诗'] }

// 杨氏之子 → 查世说新语原文 + 魏晋轶事
{ cadalMatchHints: ['世说新语', '魏晋轶事', '童趣文言'] }

// 古人谈读书（一） → 查论语 + 十三经注疏 + 儒家经典
{ cadalMatchHints: ['论语', '十三经注疏', '儒家经典'] }
```

### 匹配流程

```
用户选择篇目 → 读取 cadalMatchHints → 调用 CADAL 检索 API → 返回古籍图像/OCR文本
```

写接口的同学注意：`cadalMatchHints` 的每个元素都可以作为 CADAL 搜索的查询关键词。

## 三、三层文本难度划分规则

**文件**：`src/data/tierRules.ts`

### 三层结构

| 层级 | 对应学段 | 目标受众 | 核心理念 |
|------|----------|----------|----------|
| **C 通识层** | 第一学段 (1-2年级) | 识字启蒙阶段 | 纯白话，去掉生僻字和复杂句式，只讲画面故事 |
| **B 适配层** | 第二学段 (3-4年级) | 诗意理解阶段 | 保��原文结构，生僻字注音，简化典故 |
| **A 原版层** | 第三学段 (5-6年级) | 文言入门阶段 | CADAL 繁体原文，标注考点和古今异义 |

### 使用方式

```typescript
import { getTierRule, getTierByStage, getStageTargetAudience } from './data/tierRules';

// 获取某层规则详情
const rule = getTierRule('C');
console.log(rule.tierName);      // '通识层'
console.log(rule.stageTarget);   // '第一学段（1-2年级）'
console.log(rule.rules);         // ['去掉所有生僻字...', ...]
console.log(rule.examples);      // [{ title: '静夜思', before: '...', after: '...' }]

// 根据学段获取层级
getTierByStage(1);  // → 'C'
getTierByStage(2);  // → 'B'
getTierByStage(3);  // → 'A'

// 获取学段受众描述
getStageTargetAudience(1);  // → '1-2年级学生，具备基础识字能力...'
```

### C 层规则详情

- 去掉笔画 > 15 的生僻字
- 每句不超过 10 字
- 去掉典故和修辞（比喻除外）
- 保留画面感和故事性

### B 层规则详情

- 保留原文句式结构
- 生僻字加拼音注释
- 一句话解释典故
- 复杂长句按意群拆分

### A 层规则详情

- 保留 CADAL 繁体原文
- 标注考试高频生字
- 标注古今异义字词
- 关联课内考点标签
- 添加文言文语法注释

## 四、CADAL 资源对应关系

### 低段诗词（1-2年级）→ CADAL 蒙学/童谣/插画

| 课文类型 | CADAL 资源方向 |
|----------|---------------|
| 儿歌童诗（咏鹅、江南） | 蒙学读物、童谣古籍、古代儿童插画方志 |
| 浅近古诗（静夜思、春晓） | 唐代诗人别集、启蒙诗歌选本 |
| 自然景物（画、风、小池） | 古代山水方志、题画诗选集 |

### 中段唐宋诗词（3-4年级）→ CADAL 诗集/别集

| 课文类型 | CADAL 资源方向 |
|----------|---------------|
| 唐诗（山行、望天门山） | 《全唐诗》、唐代诗人别集 |
| 宋诗/词（饮湖上初晴后雨） | 《宋诗钞》、宋代词集 |
| 文言短篇（守株待兔） | 《韩非子》、先秦寓言集 |

### 高段文言文（5-6年级）→ CADAL 善本/刻本原版

| 课文类型 | CADAL 资源方向 |
|----------|---------------|
| 世说新语（杨氏之子） | 明清刻本《世说新语》、完整古籍善本 |
| 论语（古人谈读书） | 《十三经注疏》、儒家经典刻本 |
| 史记（田忌赛马） | 《史记》明清刻本、二十四史 |
| 其他文言（学弈、两小儿辩日） | 《孟子》《列子》善本 |

### 核心原则

```
低段 → 图像为主（蒙学插图、方志画卷）
中段 → 原文+注释（诗集、别集）
高段 → 原版图像+OCR文字（善本、刻本）
```

## 七、快速上手

### 三步走

```bash
# 1. 安装依赖
pnpm install

# 2. 启动开发服务器
pnpm dev
# → 访问 http://localhost:5173/guji-reading-system/

# 3. 构建生产版本
pnpm build
# → 输出到 dist/ 目录
```

### 改数据

所有数据文件都在 `src/data/` 下，修改后热更新即时生效：

- **加一首诗** → 编辑 `textbookDatabase.ts`，按格式新增一条
- **改难度规则** → 编辑 `tierRules.ts`
- **加异体字** → 编辑 `variantCharacters.ts`

### 构建

```bash
pnpm build       # TypeScript 类型检查 + Vite 打包
pnpm lint        # Oxlint 代码检查
```

### 部署

构建产物在 `dist/` 目录，部署到任意静态服务器即可。注意 `vite.config.ts` 中 `base` 已设为 `/guji-reading-system/`，如需修改路径前缀请同步调整。
